  Work around GWT parser's handling of empty annotations.  Currently the GWT compiler simply hardcodes the empty string for JVM properties, but this leads to the parser's "unknown annotations" list being parsed as a singleton list containing only the empty string, which is bogus.  The upshot is that a particular angular.js file has some JSDoc with a CSS example containing "@-webkit" which gets parsed as an empty annotation and breaks rhino's source position information, causing an unchecked exception and crashing the parser.
