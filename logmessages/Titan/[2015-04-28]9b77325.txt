  Make CommonsConfiguration write Durations as ms  CommonsConfiguration read Durations as unitless positive milliseconds expressed as a bare integer, but would serialize through the toString behavior used for the general case.  The toString was something like "Duration [60 ms]", so it was asymmetrical.  Special-casing Duration on the write path to make it symmetrical.  This fixes an error when copying the graph's local configuration into a Hadoop configuration object prior to launching a MapReduce-based index management job.
