  Fixed bug that prevented TAPI task execution with provider  The logic for processing Launchables was previously broken, when one or more Launchables was from a ClientProvidedBuildInvocations. Previously, this only impacted BuildInvocations obtained from providers < 1.12. However we now create client-provided Launchable instances for targeted tasks: this exposed the breakage against Gradle 2.12.  The bug was fixed by ensuring the set of Launchables is null, not empty.
