  Don't use the timestamp ordered path for name queries with counters/collections  patch by slebresne; reviewed by iamaleksey for CASSANDRA-10572  Name queries usually query sstable sequentially in recency order in the hope of only needing the more recent sstable(s). We cannot ever save sstable that way for counters and collections however.  The patch makes name queries with either counters or collections use the more generic path of looking at all sstables simultaneously.
