 [SMALLFIX] Fix NPE when getTotalBytesOnTiers is called before workerRegister  If getTotalBytesOnTiers is called after blockMaster.getWorkerId and before blockMaster.workerRegister, worker.getTotalBytesOnTiers() will be null, so the for (int i = 0; i < worker.getTotalBytesOnTiers().size(); i ++) { will throw an NPE.  To fix this we should initialize totalBytesOnTiers and similar class variables in MasterWorkerInfo to sensible defaults, e.g. empty lists/sets.
