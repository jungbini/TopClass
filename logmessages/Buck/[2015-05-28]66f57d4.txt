  [tracing] Fix built-in trace viewer in Chrome 43  Summary: `trace_viewer.js` is calling a `window.history.pushState()` API passing only one parameter.  This API takes three parameters, and at least as of Chrome 43.0.2357.65, JS raises an exception when you only pass it one.  This fixes the exception by passing all three params.
