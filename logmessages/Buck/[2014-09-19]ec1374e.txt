  pathlib: Add support to Path.match() to require entire path to match  Summary: Currently, pathlib's Path.match() has two modes:  1) Relative path mode: If neither the path nor the pattern start with `/` or a drive letter, then the pattern only needs to match a suffix of the path. This lets `*.py` match `foo/bar/baz.py`.  2) Absolute path mode: If both the path and the pattern start with `/`, then the pattern must match the entire path, so `/foo/bar/*.py` matches `/foo/bar/baz.py`.  To properly support the `excludes` feature of Buck's `glob()`, we need to allow requiring the entire relative path to match. This adds the feature, defaulting to off, and adds more tests.
