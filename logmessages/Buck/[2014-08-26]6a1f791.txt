  New class GlobalState to hold process-wide logger data  Summary: Currently, the LogFormatter class holds on to a process-wide map of <thread ID: command ID> pairs.  Now that we can have multiple concurrent clients connected, the previous assumptions that ConsoleHandler had about one console at a time existing no longer hold, which caused #4985049.  I need the <thread ID: command ID> map in ConsoleHandler, so this creates a new package-private class GlobalState which holds this process-wide map (which will be shared between LogFormatter and ConsoleHandler).
