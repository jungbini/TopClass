  Log build rule with its rule key when it is fetched from the cache.  Summary: When I look at Buck logs when debugging certain failures on CI builds, the logs only include rulekeys for which we got a cache hit, and rules that we built locally. With this change, you can find out the particular build rules associated with those cache hits, which should be helpful while debugging (especially with top down building because it's not clear from the logs which rules were skipped vs fetched from cache).
