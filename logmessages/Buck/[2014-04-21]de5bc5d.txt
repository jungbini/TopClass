  Changing an asset file under an android_resource() rule that does not include any resources should get repackaged.  Summary: This also explains the issue raised on github: https://github.com/facebook/buck/issues/100  UberRDotJavaUtil ignores any resources with empty `res` dir argument. Since the deps of `AaptPackageResources` did not include asset only resources, when an asset file changed under such rules, we wouldn't run aapt to package resources into the resulting .ap_.
