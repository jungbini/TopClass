  Fix Turkish I crash in AndroidBinaryDescription  Summary: If Buck was run on a system in Turkish locale, creating an `android_binary()` rule with `package_type = "instrumented"` would crash with `IllegalArgumentException`.  This was because we were using `String.toUpperCase()` to convert the `String` to an `Enum` without passing in a `Locale`. This would default to the system locale, which in the case of Turkish has special rules to map `i` to `\u0310`, causing a crash in `Enum.valueOf()`.  This fixes the bug and adds a test.
